DOCUMENTO DE OBJECTIVO: SISTEMA DE GESTÃO - CLÍNICA DE FISIOTERAPIA
1. VISÃO GERAL
Desenvolver um sistema web para automatizar a gestão de uma clínica de fisioterapia, integrando o controlo administrativo de estagiários com o acompanhamento clínico de pacientes.

2. OBJECTIVOS PRINCIPAIS (MVP - Mínimo Produto Viável)

Controlo de Acesso (RBAC): Implementar diferentes níveis de permissão para Gestores e Estagiários.

Gestão de Estagiários: Cadastro, monitorização e organização dos alunos em prática clínica.

Segurança de Dados: Garantir que apenas utilizadores autenticados acedam a informações sensíveis.

Interface Intuitiva: Utilizar Bootstrap para criar um ambiente de trabalho ágil e responsivo.

3. RECURSOS TÉCNICOS

Backend: Python / Django 5.x.

Frontend: HTML5, CSS3, Bootstrap 5.

Base de Dados: SQLite (Desenvolvimento).

Arquitetura: MVC (Model-View-Template).

4. ROADMAP E EXPANSÃO (Projectos Futuros)

Módulo de IA (Integração LucyAcessível-Versão clínica): O sistema foi projectado de forma modular para permitir a futura integração de um Assistente Virtual de IA.

Função: Suporte ao paciente para dúvidas sobre exercícios e agendamentos.

Tecnologia: Baseado em Processamento de Linguagem Natural (PLN).

Gestão de Prontuários: Digitalização completa das evoluções clínicas.

Por que escrever desta forma?
Separação de Preocupações: Você deixa claro que o sistema da clínica é uma coisa e a IA é outra, mas que elas podem "conversar" no futuro.

Visão de Engenheira: Ao usar termos como "MVP", "RBAC" e "Modular", você demonstra que entende como um software é planeado no mercado de trabalho.

Portfólio: No GitHub, isso mostra que você sabe planear o crescimento de um sistema sem precisar de misturar todo o código de uma vez.

Dica prática para o seu VS Code:
